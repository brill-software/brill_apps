Brill CMS Editor

On the left hand side are the web applications you have access to and also the Storybook. The upspeak web application is a demo application that you can modify.

The Storybook contains a set of components that can be dragged and dropped or copy and pasted onto pages.

Start by looking at the upspeak application and have a look at the various components in the Storybook. Also look at the CMS Help.

Creating a new application from scratch

Right click on the top level folder and select New Folder. Create a folder with the name of the application.

Under the top level folder, create directories of Database, Icons, Images, Pages, Resources, Scripts and Themes.

Version Control

Version Control provides a means of tracking changes and releasing application to the Development and Production environments.

You have a workspace in which changes are made. Your workspace is private and changes you make won't impact other people.

Your account is restricted and you can't publish changes to the develop or master branches. When you're ready for your work to be published, you need to get someone that has the cms_version_control permission to check your work and publish it. Your web application will only accessible by users when it has been checked into the master branch and deployed to the production workspace.

There might be updates to the Storybook and Demo applications. These changes will be listed below, if there are any. You can update your workspace to include the changes by clicking on the Rebase button.
